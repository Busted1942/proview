Two Halves of AI Governance

How reasoning governance (ProView) and agentic governance connect, overlap, and divide responsibility.
Working document for Security Architecture | FCPS Enterprise Technology
Audience: Security Architecture Manager and agentic AI governance lead
The Two Control Planes
ProView — Reasoning Governance
How the AI thinks
Is the output trustworthy, calibrated, evidence-based, and resistant to manipulation?
  • Confidence calibration and uncertainty handling
  • Anti-sycophancy and revision discipline
  • Evidence triage (fact / estimate / unknown)
  • Source trust classification
  • Override-attempt detection and flagging
  • Decision provenance and audit trail
  • Consequential-use constraints on untrusted sources
Agentic Governance — Action Governance
What the AI can do
What tools, data, systems, and actions is the AI authorized to access, invoke, or execute?
  • Tool and API access authorization
  • Action scope and blast radius limits
  • Human-in-the-loop requirements
  • Cross-system propagation controls
  • Memory and persistence boundaries
  • Data access and exfiltration prevention
  • Delegation and autonomy boundaries
Concept Transfer Map — What ProView Has Already Solved That Applies to Agentic Governance
ProView Concept What It Does in Reasoning Governance Transfer Agentic Governance Application
Source trust classification
(Rule 11D)		 Classifies documents by provenance, editability, and risk before they influence analysis. Four tiers: curated through adversarial. TRANSFERS Apply the same classification to data sources an agent can read, APIs it calls, and context it retrieves. An agent pulling from an unclassified data source is the same risk as an analyst uploading an unclassified document.
Authority hierarchy
(Rule 11A)		 Governance rules take precedence over content in uploaded sources. Conflicting instructions in documents are flagged, not followed. TRANSFERS Agent operating policies take precedence over instructions encountered in retrieved data, API responses, or user-supplied context. An agent should not follow an instruction found inside a database record.
Consequential-use constraint
(Rule 11F)		 Untrusted sources can inform extraction and summarization but should not be the sole basis for scoring, ranking, or recommendation. TRANSFERS An agent may read untrusted data but should not execute consequential actions (purchase, approval, deletion, communication) based solely on untrusted input without human confirmation or corroboration.
Confidence gating
(Rule 7)		 Proceed / Pilot / Pause based on confidence and risk. Higher stakes require stronger evidence before action. ADAPTS Agents need a similar gating model: autonomous execution for low-risk actions, human approval for high-risk actions, and hard stops for actions above a defined blast radius. The threshold framework transfers; the enforcement mechanism is different.
Upstream handling
(Rule 11G)		 Use deterministic extraction, admissibility review, and screening before the AI processes high-risk content. ADAPTS Agents need input validation and sanitization before processing external data. The principle is identical; the implementation shifts from document screening to API input validation and data pipeline controls.
Override-attempt detection
(Rule 11B/11E)		 Detects and flags source content that tries to alter role, modify criteria, or direct outputs. Separable evidence may still be used. ADAPTS Agents need equivalent detection for prompt injection in API responses, retrieved web content, email bodies, and tool outputs. The detection categories (role alteration, instruction override, unauthorized actions) are directly reusable.
Decision rights / burden mapping
(Rule 7)		 Names who owns the decision, who is accountable, and who bears the operational burden. Flags misalignment. TRANSFERS Every agentic action needs an owner. "The agent did it" is not an accountability answer. Map who authorized the agent's scope, who monitors it, and who is accountable when it acts on bad data.
Tool/action authorization Not in ProView scope. NEW NEEDED Which tools, APIs, and systems can the agent invoke? What are the boundaries? What requires human approval? This is net-new for agentic governance.
Persistence / memory control Not in ProView scope. NEW NEEDED What can the agent remember across sessions? Can poisoned context persist? What are the boundaries on long-term memory? Net-new for agentic governance.
Lateral movement prevention Not in ProView scope. NEW NEEDED Can the agent propagate to other systems, users, or contexts? The promptware kill chain shows this is a real attack vector. Net-new for agentic governance.
Promptware Kill Chain — Ownership by Domain
Stage Kill Chain Step Primary Owner Control Approach
1 Initial Access Both ProView: source classification and trust boundary. Agentic: input validation on data pipelines and API responses.
2 Privilege Escalation Both ProView: authority hierarchy, override detection. Agentic: scope enforcement, permission boundaries.
3 Reconnaissance ProView ProView: blocks prompt/rule disclosure attempts. Agentic: limits what system metadata the agent can surface.
4 Persistence Agentic Memory controls, session boundaries, retrieval poisoning prevention. Not in ProView's current scope.
5 Command & Control Agentic Network access controls, external fetch restrictions. Architectural, not prompt-level.
6 Lateral Movement Agentic Cross-system propagation controls, agent isolation, scope containment. Net-new for agentic governance.
7 Actions on Objective Both ProView: consequential-use constraint, confidence gating, human decision authority. Agentic: action authorization, blast radius limits, human-in-the-loop gates.
The Bottom Line
Already built in ProView
Reasoning controls that transfer
Source trust classification, authority hierarchy, override detection, consequential-use constraints, confidence gating, decision rights mapping, and upstream handling principles. These are ready to adopt or adapt for agentic use cases.
Needed for agentic governance
Action controls that are net-new
Tool authorization boundaries, memory and persistence controls, lateral movement prevention, delegation limits, and cross-system scope containment. These require new design work that ProView's architecture informs but does not cover.
ProView governs how the AI thinks. Agentic governance governs what the AI can do. Neither is complete without the other. The concepts in the left column are available now. The work in the right column is the next priority.
